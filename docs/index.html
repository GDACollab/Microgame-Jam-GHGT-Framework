globals.js

Okay, so config.ini as it currently works as stuff to be loaded in on the fly. At some point that will hopefully get actually compiled into useful data instead of being read on the fly. But I've already spent a lot of time on this system, so that improvement isn't happening this year. So we need a list of globals for any Javascript file to read easy, and for developers (i.e., you) to mess around with. I only thought of this close to finish. This is meant to be changeable for games that were developed with maybe different sizes in mind? But there's no way to guarantee compatibility. It's just to make updating portions of the code slightly easier on you.
